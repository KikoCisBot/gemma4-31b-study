🔬 Key Finding: The Benchmark Trap

95% BFCL = 0% Agent

A fine-tuned model scoring 95.50% on the Berkeley Function Calling Leaderboard scored 0/10 on our autonomous agent test — entering an infinite loop of repeated commands. The unfine-tuned base model (80% BFCL) scored higher. Standard benchmarks measure format compliance, not reasoning ability.

Bigger ≠ Better for Agents

The 33 GB Q8 model scored 5/40 while the 16 GB Q4 scored 10/40. Higher precision doesn't help if the model generates verbose responses that the agent framework can't parse efficiently.

⚙️ Methodology

Execution

Each model runs autonomously inside a Docker container (Ubuntu 24.04) with bash, file I/O, and web search tools. No human intervention. 10-minute timeout per challenge.

Scoring v2.1

Deep validation: checks actual data values (not keywords), verifies configs are structurally valid, penalizes anti-patterns (loops, error blindness). score_challenge.py is open source.

Reproducibility

All scripts, Docker configs, and scorers available at HuggingFace. Run it on your own models.

Variance

LLM outputs are stochastic. We run multiple evaluations where possible. Opus stabilized at 26/40 across 2 runs. Local models run once due to cost constraints.
