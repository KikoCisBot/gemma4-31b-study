Key findings

  • Unsloth UD-IQ3_XXS (11.84 GB) beats f16 on Char-F1 — the dynamic per-layer bit allocation + Gemma-specific imatrix preserves structure better than the full-precision model itself on this benchmark.
  • This repo's custom IQ2_M (10.17 GB, CLI-tuned imatrix) matches f16 Char-F1 (84.71 vs 84.76) and beats f16 on BLEU-4 (22.39 vs 21.02) at 6× smaller. Beats Unsloth UD-IQ2_M on every metric at 0.6 GB smaller — evidence that domain-focused calibration data pushes the 2.8 BPW Pareto frontier further. [HF repo →]
  • UD-IQ2_M at 10.75 GB matches f16 quality (F1 84.02% vs 84.76%). +25.4 F1 points over standard Q2_K at identical 2.70 BPW — a massive gap caused entirely by the calibration method.
  • Q2_K collapses on Gemma 4 (F1 58.6%, outputs "-y-y-y-y"). Standard scalar 2-bit quantization is not viable for this architecture's wide logit distribution + RMSNorm scaling.
  • Layer pruning is dominated by dynamic quantization. Otter-v3 (58 layers, 12.1 GB) scored 6 F1 lower than the custom IQ2_M (60 layers, 10.17 GB). Pruning without recovery fine-tuning loses more than it saves.

02Code & Agentic Benchmarks (IQ2_M, 10.17 GB)

Beyond CLI-style NL2Bash, we tested the custom IQ2_M on three code and agentic benchmarks. The result: a 10 GB quantized 31B matches or beats many f16 models in its tier on coding and function-calling tasks — using the same weights that win NL2Bash.

Results summary

Benchmark Split Score Passed / Total Failure classes (top)
HumanEval+ Full (164) 88.41% pass@1 145 / 164 wrong_output (15), missing_import (3)
MBPP+ Full (378) 82.01% pass@1 310 / 378 wrong_output (35), missing_import (28)
BFCL v3 simple Full (400) 92.25% accuracy 369 / 400 args_mismatch (30), wrong_name (1)

What it means

  • 88.4% HumanEval+ at 2.84 BPW is strong for a 10 GB quantized model. Only 19 of 164 problems failed, almost all from subtle logic (wrong_output: off-by-one, edge case) — not catastrophic cuantización damage.
  • 92.25% BFCL simple (single function-call) shows the quantization preserves function-signature comprehension and argument extraction. Only 30 of 400 had argument mismatches and just 1 picked the wrong function name.
  • MBPP+ weak spot: 28 of 68 failures (41%) are missing_import_or_name — the model writes code using math.sqrt or re.match without the import statement. Fixable by domain-specific calibration.
  • Context for public numbers: Google reports Gemma 4 31B f16 at 80.0% on LiveCodeBench v6 and 76.9% on Tau2. HumanEval+/MBPP+/BFCL scores for Gemma 4 31B are not officially published — this appears to be the first public evaluation on these benchmarks.

Next step — failure-driven training

The 118 failure cases are the seed of a domain-mixed importance matrix plus a LoftQ-style LoRA delta trained on what the model gets wrong. Goal: preserve or beat f16 on these benchmarks with a smaller footprint. Full plan and anti-overfit strategy (SWE-bench + LiveCodeBench validation) in the study repo.

03Layer Ablation Study

For every one of the 60 transformer layers we measured Block Influence (BI), logit-lens decision point, and the perplexity delta when the layer is individually zeroed out. Surprise: several middle layers have negative deltas — removing them improves the model.

Single-layer ablation: perplexity delta %

Positive bars (red) = layer is critical (breaks model when removed). Negative bars (green) = removing this layer improves perplexity.

Block Influence per layer (log scale)

Layer findings

  • Layer 27 & 29 are net-negative: single ablation improves PPL by -74% and -71%. These layers are adding noise to the residual stream.
  • Layers 0, 1, 58, 59 are critical: ablation explodes PPL ×100 to ×2360. Never remove them.
  • Decision happens in layers 56-59: logit-lens shows the target token rank collapses from 2533 → 51 → 3 → 1 in the last 4 layers.
  • Middle layers (10-22) are almost inert: BI < 0.005 across all 6 probe categories, but compound effects prevent dropping more than 2 at once without fine-tuning recovery.

05Methodology

Everything is reproducible — code and raw measurements are public.

Layer study pipeline

phase_a.py  # Block Influence: ||h_out - h_in||, cosine(h_in, h_out)
phase_b.py  # Logit Lens: project h_l through norm+lm_head for each layer
phase_c.py  # Single-layer ablation: skip one layer, measure PPL delta
phase_c2.py # Block ablation: drop N layers together, measure PPL

Benchmark setup

nl2bash_bench_sub.py \
  --model gguf/X.gguf \
  --limit 50 --max-tokens 200 \
  --report-every 5 --out bench.jsonl

Hardware

Apple M4 Max, 128 GB unified memory. Full GPU offload via Metal. All measurements single-run, deterministic at temp=0.1 (small residual variance).
