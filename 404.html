The Specialization Boundary: Where Large Language Models Encode Domain Knowledge and Why Quantization Breaks It

Kiko Cisneros & Claude Opus 4.6
Independent Research · Utopia IA · April 2026
Abstract. We present a per-dimension activation analysis of Google's Gemma 4 31B Instruct across 9 task categories (code, tool-calling, multilingual, math, creative). We identify layer 18 as a phase transition: below it, task categories share 70%+ of their active dimensions (Jaccard ≥ 0.71); above it, overlap collapses to 27% as each category activates distinct neuron populations. This "specialization boundary" explains an empirical puzzle: why 1.75 BPW quantization (IQ1_M) destroys code generation (HumanEval+ 21%) while preserving tool-calling (BFCL 87%) — the tool-calling circuit resolves earlier in shared layers that survive aggressive compression. We validate across 6 model variants (10–35 GB) on 4 benchmarks, and show that domain-focused imatrix calibration can shift the Pareto frontier by targeting preservation at the specialization boundary.

1The Specialization Boundary at Layer 18

For each of the 60 transformer layers, we measure which of the 5,376 hidden dimensions activate most strongly for each task category (top-200 per category). The Jaccard index between two categories' active dimensions tells us how much they share the same neural circuitry.

Jaccard overlap vs layer depth

Each line is a pair of categories. The drop at layer 18 is the specialization boundary.

Finding 1. Between layers 15 and 18, the Jaccard overlap between python_code and tool_calling drops from 0.71 to 0.27 — a 62% collapse in 3 layers. Every category pair shows the same phase transition. Below layer 18, the model uses shared general-purpose circuits. Above it, each task type lights up its own neuron population.

2Per-Category Neuron Specialization

At each sampled layer, we count how many of a category's top-200 dimensions are unique to it (not in any other category's top-200). More unique dims = more specialized circuitry.

Unique dimensions per category vs layer

Higher = this category uses neurons that no other category uses at this layer.

Finding 2. Tool-calling has the most unique dimensions in early layers (3–9) AND mid layers (18–24), confirming it uses dedicated circuits from the very start. Code and math share almost everything until layer 18, then diverge. Languages (Spanish, Chinese) have unique embedding dimensions (layer 0) but reconverge with English by layer 12.

3Why Quantization Breaks Code but Preserves Tool-Calling

At 1.75 BPW (IQ1_M), each weight is represented by ~1.75 bits. The quantization error is distributed across all dimensions uniformly. But the impact of that error depends on whether the corrupted dimensions are shared (low impact — redundancy absorbs it) or specialized (high impact — no fallback).

The IQ1_M puzzle, explained. At 7.4 GB, HumanEval+ collapses to 21% while BFCL stays at 87%. Our region analysis shows why: code generation relies on specialized dimensions in layers 18–39 (Jaccard 0.27 with other tasks), which are destroyed by uniform 1.75 BPW quantization. Tool-calling uses shared dimensions in layers 0–15 (Jaccard 0.71+) plus a small set of dedicated early-layer neurons that survive because they're reinforced by cross-category redundancy.
Finding 3. The practical implication: to quantize below 2 BPW without losing code, you must protect the specialization boundary (layers 18–24) with higher bit budgets. A uniform bit allocation treats shared and specialized dimensions equally — which is provably suboptimal given the phase transition we observe.

6Methodology

Activation analysis

50 prompts across 9 categories (code, tool-calling, multilingual, math, creative writing), processed through Gemma 4 31B IT (4-bit MLX quantization on M4 Max). For each of 25 sampled layers, we record the per-dimension absolute delta |hout − hin| averaged across sequence positions and prompts within each category. Top-200 dimensions per category per layer are compared via Jaccard index.

Benchmarks

HumanEval+ (164 problems, pass@1), MBPP+ (378 problems, pass@1), BFCL v3 simple (400 function-call problems, exact-match accuracy), NL2Bash (50 examples, Char-F1). All via llama-cli subprocess, reasoning off, temp 0.1, ctx 4096.

Hardware

Apple M4 Max, 128 GB unified memory. Full GPU offload via Metal.
